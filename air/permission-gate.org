#+title: Permission Gate
#+state: complete
#+FILETAGS: :extension:feature:

* Summary

A configurable safety gate that intercepts bash tool calls and prompts for confirmation before executing dangerous commands. Selecting "No" opens an inline editor to provide a free-text rejection reason that is fed back to the assistant.

Based on pi's built-in [[https://github.com/badlogic/pi-mono/blob/main/packages/coding-agent/examples/extensions/permission-gate.ts][permission-gate example]] and [[https://github.com/rytswd/pi-agent-extensions/pull/13][PR #13 by Mic92]].

* Motivation

The built-in example only blocks 3 bash patterns (=rm -rf=, =sudo=, =chmod 777=). Real-world use needs more patterns, the ability to customise them per project, and free-text rejection reasons so the assistant can adjust (as discussed in [[https://github.com/rytswd/pi-agent-extensions/issues/4][#4]]).

slow-mode already gates write/edit, but permission-gate serves a different purpose: it's always-on protection against dangerous commands, not a review workflow. They complement each other.

** Goals
- Configurable pattern lists — users can add/remove rules via =/gate add|rm= or by editing config files.
- Project-local overrides — different projects have different danger zones.
- Free-text rejection reason — tell the assistant /why/ the command was blocked.
- Clear labels — show WHY a command was flagged ("force push", "recursive delete").
- Status bar indicator — = gate= shown in statusline.
- Toggle — =/gate= to enable/disable.
- Fallback safety — if all user rules fail to compile, defaults are restored automatically.
- Non-interactive safety — block by default when no UI available (headless/print mode).
- Desktop notification — emits =permission-gate:waiting= event consumed by notify extension.

** Non-Goals
- Replacing slow-mode for write/edit review.
- Gating every bash command (removed — too noisy, use =/gate add= for specific patterns instead).
- Sandboxing or filesystem isolation.

* Proposal

** Default Rules

| Category | Pattern | Label |
|----------+-------------------------------------------+-----------------------------|
| Delete | =rm -rf=, =rm --recursive= | recursive delete |
| Privilege | =sudo= | sudo |
| Permissions | =chmod 777= | world-writable permissions |
| Device | => /dev/sd*= | raw device redirect |
| Git destructive | =git push -f=, =git reset --hard= | force push, hard reset |
| Git clean | =git clean -f= | git clean |
| Git restore | =git checkout .=, =git restore= | discard changes |
| Network exec | =curl \| sh=, =wget \| sh= | pipe to shell |
| GitHub CLI | =gh repo (create\vert{}delete\vert{}archive)= | modify GitHub repo |
| GitHub CLI | =gh release (create\vert{}delete)= | modify GitHub release |

** Configuration

=~/.config/pi-agent-extensions/permission-gate/rules.json= (see =.ref/config-dir.org=):

#+begin_src json
{
  "extraRules": [
    { "pattern": "\\bdocker\\s+rm\\b", "label": "docker remove" },
    { "pattern": "\\bterraform\\s+destroy\\b", "label": "terraform destroy" }
  ],
  "disabledRules": ["sudo"]
}
#+end_src

| Field | Description |
|---------------+----------------------------------------------|
| =rules= | Override the default rule set entirely (replaces all built-ins) |
| =extraRules= | Add rules on top of defaults |
| =disabledRules= | Disable specific rules by label |

Each rule entry:
| Field | Description |
|----------+--------------------------------------|
| =pattern= | Regex string (compiled with =new RegExp=) |
| =label= | Short name shown in the prompt |
| =flags= | Optional regex flags (default: ="i"= for case-insensitive) |

Project-local overrides at =<cwd>/.pi/permission-gate.json= — merged on top of global config:

#+begin_src json
{
  "extraRules": [
    { "pattern": "\\bkubectl\\s+delete\\b", "label": "kubectl delete" }
  ]
}
#+end_src

Rules can also be managed interactively via =/gate add= and =/gate rm=, which write to the global config file.

** Commands

| Command | Alias | Description |
|-----------------+-------+---------------------------------------------|
| =/gate= | | Toggle on/off |
| =/gate list= | =ls= | Show active rules (grouped by source) |
| =/gate add= | | Add a rule interactively (pattern + label) |
| =/gate remove= | =rm= | Remove a rule by label |

** Interaction Flow

#+begin_example
Agent calls bash: git push --force origin main

  ⚠️ Dangerous command (force push)
  git push --force origin main

  > Yes
    No

  ↑↓ • Enter • Esc block
#+end_example

Selecting "No" opens an inline editor:

#+begin_example
  > Yes
    No ✎

  Reason:
  ┌──────────────────────────────────────────────────────┐
  │ use the wrapper script instead                       │
  └──────────────────────────────────────────────────────┘

  Enter submit • Esc back
#+end_example

The assistant then sees: =Blocked by user (force push): use the wrapper script instead=

** Relationship to slow-mode

| Feature | permission-gate | slow-mode |
|---------+---------------------------------+---------------------------------|
| Tools | bash only | write, edit only |
| Purpose | Safety catch | Review workflow |
| Display | Yes/No + reason editor | Full diff view |
| When | Specific patterns match | Every write/edit |
| Default | On | Off |

Both can run simultaneously. permission-gate fires first (earlier in =package.json= extension list).

** Events

| Event | When | Consumer |
|------------------------------+----------------------------+---------------------|
| =permission-gate:waiting= | Prompt shown to user | notify (desktop ping) |
| =permission-gate:resolved= | User made yes/no decision | _(available for use)_ |

** Env Var

=PI_NO_GATE=1= to disable at startup (for subagent sessions via chronoa).

* Implementation Notes

** Error Handling

- Invalid JSON in config file → warns user, uses empty config (defaults apply).
- Invalid regex in a single rule → warns user, skips that rule, others still work.
- All rules fail to compile → falls back to built-in defaults with a warning.

** Config Directory

Follows the convention in =.ref/config-dir.org=. Config at =~/.config/pi-agent-extensions/permission-gate/rules.json=.
